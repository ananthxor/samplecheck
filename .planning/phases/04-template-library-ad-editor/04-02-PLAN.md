    **Step 1: Create `creatives-api.ts`** at `apps/web/src/features/creatives/api/creatives-api.ts`:

    Thin wrapper around Supabase client calls for the `creatives` table. Import `supabase` from `@/lib/supabase` and types from `@scrolltoday/shared`.

    Functions:
    - `fetchCreatives()`: SELECT * ordered by updated_at DESC. Returns `Tables<'creatives'>[]`.
    - `fetchCreativeById(id: string)`: SELECT * WHERE id, single. Returns `Tables<'creatives'>`.
    - `createCreative(creative: Insertable<'creatives'>)`: INSERT with .select().single(). Returns created row.
    - `updateCreative(id: string, updates: Updatable<'creatives'>)`: UPDATE WHERE id with .select().single(). Returns updated row.
    - `deleteCreative(id: string)`: DELETE WHERE id. Returns void.
    - `fetchCreativeByToken(token: string)`: SELECT * WHERE preview_token = token, single. Returns row. (This will be used in Plan 03 for public preview; include it now so the API module is complete.)

    All functions throw on error with `new Error(error.message)`.

    **Step 2: Create `use-creatives.ts`** at `apps/web/src/features/creatives/hooks/use-creatives.ts`:

    TanStack Query hooks wrapping the API:
    - `useCreatives()`: `useQuery({ queryKey: ['creatives'], queryFn: fetchCreatives })`
    - `useCreative(id: string | undefined)`: `useQuery({ queryKey: ['creatives', id], queryFn: () => fetchCreativeById(id!), enabled: !!id })`
    - `useCreateCreative()`: `useMutation({ mutationFn: createCreative, onSuccess: () => queryClient.invalidateQueries({ queryKey: ['creatives'] }) })`
    - `useUpdateCreative()`: `useMutation({ mutationFn: ({ id, updates }) => updateCreative(id, updates), onSuccess: (_, { id }) => queryClient.invalidateQueries({ queryKey: ['creatives'] }) })`
    - `useDeleteCreative()`: `useMutation({ mutationFn: deleteCreative, onSuccess: () => queryClient.invalidateQueries({ queryKey: ['creatives'] }) })`

    Import `useQuery`, `useMutation`, `useQueryClient` from `@tanstack/react-query`.

    **Step 3: Create `renderer.ts`** at `apps/web/src/features/editor/lib/renderer.ts`:

    The core HTML generation function that produces self-contained HTML for the iframe `srcdoc`.

    `generatePreviewHtml(config: TemplateConfig): string` -- returns a complete HTML document string.

    The HTML document includes:
    - DOCTYPE, html, head with charset, viewport meta, and base styles (reset, system-ui font, overflow hidden)
    - A `<div id="creative-root">` container
    - A `<script>` block that:
      1. Stores initial config from `JSON.stringify(config)` (XSS-safe: sanitize by escaping `</script>` in stringified output)
      2. Defines a `render(cfg)` function that calls format-specific renderers based on `cfg.type`
      3. Implements `renderStaticBanner(cfg)`: div with background color, centered content, optional image, headline (h1), bodyText (p), CTA button (a tag styled inline)
      4. Implements `renderMultiFrame(cfg)`: auto-rotating frames using setInterval at cfg.frameDuration; shows one frame at a time with fade transition; each frame has headline, bodyText, image, background color
      5. Implements `renderInFeed(cfg)`: native-style card with sponsor logo/name, headline, image, body text, CTA
      6. For all other format types: renders a placeholder `<div>` showing "Preview for {type}" centered with muted styling -- these renderers will be implemented in Phases 5/6
      7. Listens for `message` events: when `event.data.type === 'CONFIG_UPDATE'`, calls `render(event.data.payload)` for incremental DOM updates
      8. Calls `render(currentConfig)` on initial load

    IMPORTANT: Escape `</script>` tags inside JSON.stringify output by replacing `</` with `<\\/` to prevent premature script tag closure. Also escape any HTML-special chars in text content using a simple sanitize helper (replace `<`, `>`, `&`, `"`, `'`).

    **Step 4: Create `preview-message.ts`** at `apps/web/src/features/editor/lib/preview-message.ts`:

    Type definitions and helpers for the postMessage protocol between editor and iframe:
    ```typescript
    export type PreviewMessage =
      | { type: 'CONFIG_UPDATE'; payload: TemplateConfig }

    export function sendConfigUpdate(iframe: HTMLIFrameElement, config: TemplateConfig): void {
      iframe.contentWindow?.postMessage({ type: 'CONFIG_UPDATE', payload: config }, '*')
    }
    ```

    **Step 5: Create `use-editor-state.ts`** at `apps/web/src/features/editor/hooks/use-editor-state.ts`:

    Custom hook managing the editor's mutable state. Page-scoped (not global).

    Parameters: `initialConfig: TemplateConfig, creativeId?: string` (creativeId is set when editing existing creative).

    State:
    - `config`: current TemplateConfig (useState, initialized from initialConfig)
    - `isDirty`: boolean tracking unsaved changes (compare config to last saved config via JSON.stringify)
    - `isSaving`: boolean for save-in-progress state
    - `selectedSize`: `{ width: number, height: number }` for the currently selected creative size

    Returns: `{ config, setConfig, updateConfig, isDirty, isSaving, setIsSaving, selectedSize, setSelectedSize, resetDirty }`

    `updateConfig(partial: Partial<TemplateConfig>)`: merges partial into current config using spread. This is used by the form's `onChange` to update individual fields without replacing the whole config.

    **Step 6: Create `use-image-upload.ts`** at `apps/web/src/features/editor/hooks/use-image-upload.ts`:

    Hook for uploading images to Supabase Storage `creative-assets` bucket.

    Uses `useAuth()` to get `profile.advertiser_id`. Uses `supabase.storage.from(STORAGE_BUCKET)` (import STORAGE_BUCKET from `@scrolltoday/shared`).

    `uploadImage(file: File, creativeId: string)`:
    1. Validate advertiser_id exists
    2. Generate unique path: `${advertiser_id}/${creativeId}/${crypto.randomUUID()}.${ext}`
    3. Upload with `cacheControl: '31536000'`, `upsert: false`
    4. On success: get public URL via `getPublicUrl(filePath)`
    5. Return `{ url: publicUrl, path: filePath }` or null on error

    State: `isUploading`, `error`. Returns: `{ uploadImage, isUploading, error }`.

    For new creatives that don't have an ID yet, use a temporary UUID generated at editor initialization. The actual creative ID is set on first save.

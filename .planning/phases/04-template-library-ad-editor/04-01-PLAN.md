    - `animatedBannerConfigSchema`: headline, bodyText, ctaText, backgroundColor, textColor, ctaColor, imageUrl, animationType (z.enum ['fade', 'slide', 'bounce', 'zoom'] default 'fade')
    - `countdownConfigSchema`: targetDate (z.string()), headline, bodyText, ctaText, ctaUrl, backgroundColor, textColor
    - `videoEndcardConfigSchema`: videoUrl, endcardHeadline, endcardBodyText, endcardImageUrl, ctaText, ctaUrl, autoplay boolean default false
    - `clickToPlayConfigSchema`: videoUrl, thumbnailImageUrl, headline, ctaText, ctaUrl

    - Middle: Template name (font-semibold), format badge (using shadcn Badge component), and description (text-muted-foreground, text-sm, line-clamp-2)
    - Bottom: Size badges showing available sizes (e.g., "728x90", "300x250") as small muted badges
    - Whole card is clickable via `onSelect`
    - Use shadcn Card component (already installed from Phase 3)
    - Hover state: slight shadow elevation (shadow-md transition)

    **Step 2: Create `template-filters.tsx`**

    Filter controls for ad type and format. Props: `selectedType: string | null, selectedFormat: string | null, onTypeChange: (type: string | null) => void, onFormatChange: (format: string | null) => void`.

    Layout:
    - Row with two shadcn Select dropdowns: "Ad Type" and "Format"
    - "Ad Type" options: "All Types" + entries from `AD_TYPES` (import from `features/dashboard/data/ad-types`)
    - "Format" options: dynamic based on selected type. If no type selected, show all 14 formats. If type selected, show only formats for that type.
    - When type changes, reset format to null
    - Include a clear filters button/link when any filter is active

    **Step 3: Create `template-grid.tsx`**

    Responsive grid that renders filtered templates. Props: `templates: Template[], onSelect: (template: Template) => void`.

    Layout:
    - CSS grid: 1 col on mobile, 2 cols on md, 3 cols on lg, 4 cols on xl
    - Gap-6 between cards
    - Empty state: if no templates match filters, show "No templates found for this filter" message

    **Step 4: Create `templates-page.tsx`**

    The templates browsing page. Reads `type` and `format` query params from URL (via `useSearchParams`). Filters templates from the registry accordingly. Passes filtered templates to TemplateGrid.

    - Read `type` and `format` from `useSearchParams()`
    - Initialize filter state from URL params
    - When filters change, update URL params (via `setSearchParams`)
    - When a template card is clicked, navigate to `/creatives/new?template={template.id}` (editor route, built in Plan 02)
    - Page title: "Choose a Template" with subtitle showing count of matching templates
    - Below title: TemplateFilters component
    - Below filters: TemplateGrid

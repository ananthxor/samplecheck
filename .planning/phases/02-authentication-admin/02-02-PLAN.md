**Step 2: Create `apps/web/src/contexts/auth-context.tsx`**

AuthProvider component that wraps the entire app. Manages:
- `user: User | null` -- Supabase auth user
- `profile: Tables<'user_profiles'> | null` -- user profile from DB
- `isLoading: boolean` -- true until initial session check completes
- `isAdmin: boolean` -- derived from `profile?.role === 'super_admin'`
- `mustChangePassword: boolean` -- derived from `profile?.must_change_password`

Implementation:
1. Create `AuthContext` with `createContext<AuthContextType | null>(null)`
2. In `AuthProvider`, use `useState` for the auth state object
3. Create `fetchProfile(userId: string)` using `useCallback` -- queries `user_profiles` where `auth_user_id = userId`, returns single row
4. Set up `supabase.auth.onAuthStateChange` in a `useEffect`:
   - CRITICAL: Keep the callback synchronous. Do NOT await inside it.
   - On session present: set user from `session.user`, then use `setTimeout(() => { fetchProfile... }, 0)` to dispatch async profile fetch OUTSIDE the callback. This prevents the deadlock described in research (Pitfall 3).
   - On no session: set user/profile to null, isLoading to false
   - Cleanup: return `subscription.unsubscribe()`
5. Expose auth actions:
   - `signIn(email, password)`: calls `supabase.auth.signInWithPassword({ email, password })`. Returns `{ error }`. On success the `onAuthStateChange` listener handles state update automatically.
   - `signOut()`: calls `supabase.auth.signOut()`. The listener handles cleanup.
   - `updatePassword(newPassword)`: calls `supabase.auth.updateUser({ password: newPassword })`. On success, also updates `user_profiles` to set `must_change_password = false` for the current user. Returns `{ error }`.
6. Export `AuthProvider` component and `useAuth()` hook

Import types from `@supabase/supabase-js` for `User` and from `@scrolltoday/shared` for `Tables`.

**Step 3: Create `apps/web/src/hooks/use-auth.ts`**

This is a convenience re-export so consumers can import from `@/hooks/use-auth` instead of `@/contexts/auth-context`.

**1. `apps/web/src/features/auth/components/protected-route.tsx`**

Two guard components using `useAuth()` + `Navigate` + `Outlet` from `react-router`:

`ProtectedRoute`:
- If `isLoading`, render a centered spinner/loading text (simple `<div className="min-h-screen flex items-center justify-center"><p>Loading...</p></div>`)
- If no `user`, `<Navigate to="/login" replace />`
- If `mustChangePassword` AND current path is NOT `/change-password`, `<Navigate to="/change-password" replace />`
- Otherwise, `<Outlet />`

`AdminRoute`:
- If `isLoading`, render loading
- If no `user`, `<Navigate to="/login" replace />`
- If not `isAdmin`, `<Navigate to="/" replace />`
- Otherwise, `<Outlet />`

Use `useLocation()` from `react-router` to check current path for the must-change-password redirect.

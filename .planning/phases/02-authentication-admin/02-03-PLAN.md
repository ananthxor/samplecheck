**1. `apps/web/src/features/admin/api/admin-api.ts`**

Type-safe wrappers for Edge Function calls using `supabase.functions.invoke()`:

```typescript
interface CreateUserPayload {
  email: string
  password: string
  display_name?: string
  advertiser_name: string
}

interface UserProfile {
  id: string
  auth_user_id: string
  role: 'super_admin' | 'advertiser'
  display_name: string | null
  must_change_password: boolean
  advertiser_id: string | null
  advertisers: { id: string; name: string; contact_email: string | null } | null
}

interface AdminUser {
  id: string
  email: string
  created_at: string
  last_sign_in_at: string | null
  profile: UserProfile | null
}
```

Export three async functions:
- `adminCreateUser(payload: CreateUserPayload)`: invokes `admin-create-user`, throws on error, returns `{ user_id, advertiser_id, email }`
- `adminListUsers()`: invokes `admin-list-users`, returns `{ users: AdminUser[] }`
- `adminResetPassword(userId: string, newPassword: string)`: invokes `admin-reset-password`, returns `{ success, user_id }`

IMPORTANT: `supabase.functions.invoke()` returns `{ data, error }`. The `data` is already parsed JSON when the response Content-Type is application/json. Check for `error` first. Also check if `data?.error` exists (Edge Functions may return 400 with `{ error: message }` in the body, but `functions.invoke` may not treat non-2xx as errors in all cases). Handle both paths.

**2. `apps/web/src/features/admin/components/user-list.tsx`**

- Style with Tailwind: clean table with `divide-y`, hover rows, responsive. Use `overflow-x-auto` wrapper for mobile.

**3. `apps/web/src/features/admin/components/create-user-dialog.tsx`**

A dialog/modal for creating new users. Props: `open: boolean`, `onClose()`, `onCreated()`.
- Use a simple modal overlay pattern (no shadcn dialog -- keep it pure Tailwind for now to avoid adding shadcn components before Phase 3):
  - Fixed overlay with `bg-black/50`, centered white card with `max-w-md`
  - Close on backdrop click or Escape key
- Form fields: email (required), password (required, min 8 chars), display_name (optional), advertiser_name (required)
- Auto-generate a random initial password (16 chars, alphanumeric) in a readonly field with a "Copy" button. Admin can override if desired.
- Submit button with loading state
- On submit: call `adminCreateUser(payload)`. On success: show toast, call `onCreated()`, close dialog. On error: show error inline.
- Show the generated password prominently after successful creation with a "Copy to clipboard" button and a note: "Save this password -- it cannot be retrieved later."

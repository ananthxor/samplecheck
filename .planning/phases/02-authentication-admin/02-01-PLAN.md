  <action>
Create shared utilities and three Edge Functions for admin user management. All functions use Deno runtime with ESM imports from `https://esm.sh/@supabase/supabase-js@2`.

**1. `supabase/functions/_shared/cors.ts`**
Export `corsHeaders` object with:
- `Access-Control-Allow-Origin: *`
- `Access-Control-Allow-Headers: authorization, x-client-info, apikey, content-type`

**2. `supabase/functions/_shared/supabase-admin.ts`**
Export two helper functions:
- `createAdminClient()`: Creates Supabase client with `SUPABASE_SERVICE_ROLE_KEY` from `Deno.env`. Do NOT pass any Authorization header to this client -- the service_role key IS the auth. Returns the admin client.
- `verifySuperAdmin(req: Request)`: Takes the incoming request, creates a SEPARATE Supabase client using `SUPABASE_ANON_KEY` + the caller's Authorization header, calls `getUser()` to get the caller's ID, then uses the admin client to query `user_profiles` where `auth_user_id = caller.id` and `role = 'super_admin'`. Returns `{ caller, adminClient }` on success, throws `Unauthorized` or `Forbidden` errors on failure. CRITICAL: Use TWO separate clients -- one with service_role (for admin ops), one with caller JWT (for auth verification). Never pass caller's JWT to the service_role client.

**3. `supabase/functions/admin-create-user/index.ts`**
POST endpoint. Accepts JSON body: `{ email, password, display_name?, advertiser_name }`.
Steps:
1. Handle OPTIONS with corsHeaders
2. Call `verifySuperAdmin(req)` to get `adminClient`
3. `auth.admin.createUser({ email, password, email_confirm: true })` -- email_confirm:true means no verification email sent
4. Insert into `advertisers` table: `{ name: advertiser_name, contact_email: email }`
5. Insert into `user_profiles` table: `{ auth_user_id: newUser.id, advertiser_id: advertiser.id, role: 'advertiser', display_name, must_change_password: true }`
6. If any step after createUser fails, attempt cleanup: `auth.admin.deleteUser(newUser.id)` and delete the advertiser if created
7. Return 201 with `{ user_id, advertiser_id, email }`
8. On error, return appropriate status (401/403/400) with `{ error: message }`
All responses include corsHeaders + Content-Type: application/json.

**4. `supabase/functions/admin-list-users/index.ts`**
GET endpoint (also responds to POST for flexibility with `functions.invoke`).
Steps:
1. Handle OPTIONS with corsHeaders
2. Call `verifySuperAdmin(req)` to get `adminClient`
3. `auth.admin.listUsers()` to get all auth users
4. Query `user_profiles` with `select('*, advertisers(*)')` using adminClient to get profiles with advertiser data
5. Merge: for each auth user, find matching profile by `auth_user_id`
6. Return 200 with `{ users: mergedArray }` where each user has: id, email, created_at, last_sign_in_at, profile (with role, display_name, must_change_password, advertiser info)

**5. `supabase/functions/admin-reset-password/index.ts`**
POST endpoint. Accepts JSON body: `{ user_id, new_password }`.
Steps:
1. Handle OPTIONS with corsHeaders
2. Call `verifySuperAdmin(req)` to get `adminClient`
3. `auth.admin.updateUserById(user_id, { password: new_password })` to set the new password
4. Return 200 with `{ success: true, user_id }`

Validate required fields in each function body. Return 400 with descriptive error for missing fields.

# Plan 10-07: Engagement Breakdown & Multi-Series Chart (GSC Style)

**Goal:** Implement a Google Search Console-style analytics dashboard that allows multi-metric selection and provides a detailed breakdown of engagement types (flips, swipes, scratches, etc.).

## Context
Currently, engagements are aggregated into a single number. Users want to see specific interaction types and compare multiple metrics (e.g., Impressions vs. Clicks) on the same chart simultaneously.

## Tasks

### 1. Database Schema Upgrade
- Create a new migration to add `engagement_metrics` JSONB column to `daily_metrics`.
- Update `rollup_today_metrics` and `rollup_daily_metrics` functions to aggregate interaction types from `ad_events.extra_data->>'type'` into a JSONB object (e.g., `{"flip": 12, "swipe": 5}`).

### 2. Frontend Type & API Updates
- Update `DailyMetricRow` and `ChartDataPoint` interfaces in `analytics-types.ts` to include `engagement_metrics`.
- Update `aggregateByDate` and `aggregateSummary` to handle merging of the new JSONB metrics.
- Extend `METRIC_LABELS` and `MetricKey` to support dynamic interaction metrics.

### 3. Google Search Console Style Chart
- Refactor `MetricsChart.tsx` to:
  - Replace `Tabs` with a multi-select toggle system (clickable cards or badges).
  - Update `AreaChart` to render an `<Area />` for each selected metric.
  - Implement dynamic Y-Axis scaling or multiple Y-Axes if units differ (e.g., count vs. percentage).
  - Ensure the legend and tooltip clearly distinguish multiple series.

### 4. Verification
- Generate specific interactions (flip, swipe, scratch).
- Trigger rollup and verify the JSONB column contains correct counts.
- Verify the chart renders multiple lines when multiple metrics are toggled.

## Technical Details

### SQL Aggregation logic:
```sql
-- Conceptual aggregation of engagement types
jsonb_object_agg(interaction_type, interaction_count)
-- via:
(
  SELECT jsonb_object_agg(t, count)
  FROM (
    SELECT extra_data->>'type' as t, count(*) as count
    FROM ad_events
    WHERE event_type = 'engagement' -- and other filters
    GROUP BY extra_data->>'type'
  ) s
)
```

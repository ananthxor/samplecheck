1. Create `apps/web/src/features/analytics/lib/analytics-types.ts`:
   - `ChartDataPoint` interface: `{ date: string, impressions: number, clicks: number, ctr: number, engagements: number, viewableImpressions: number, avgDwellTimeMs: number }`.
   - `DateRangePreset` type: `'7d' | '30d' | '90d' | 'this-month' | 'last-month'`.
   - `DATE_PRESETS` array: `[{ label: 'Last 7 days', value: '7d' }, { label: 'Last 30 days', value: '30d' }, { label: 'Last 90 days', value: '90d' }, { label: 'This month', value: 'this-month' }, { label: 'Last month', value: 'last-month' }]`.
   - `getDateRange(preset: DateRangePreset): { start: string; end: string }` — uses native Date API (no date-fns). Today = `new Date().toISOString().split('T')[0]`. For '7d': start = today - 6 days. For 'this-month': start = first of current month. For 'last-month': start = first of previous month, end = last day of previous month.
   - `AnalyticsFilters` interface: `{ creativeId?: string; campaignId?: string }`.
   - `aggregateByDate(rows: DailyMetricRow[]): ChartDataPoint[]` — groups rows by metric_date, sums impressions_served/clicks/engagements/impressions_viewable/total_dwell_time_ms, computes CTR = (clicks/impressions)*100 rounded to 2 decimals, computes avgDwellTimeMs = total_dwell_time_ms / max(engagements, 1). Returns sorted by date.
   - `aggregateSummary(rows: DailyMetricRow[]): { impressions: number, clicks: number, ctr: number, engagements: number, viewableImpressions: number, totalDwellTimeMs: number }` — sums all rows into totals. totalDwellTimeMs is the sum of all total_dwell_time_ms values across rows.
   - Import `DailyMetricRow` type from analytics-api (or define it here if circular; prefer defining here and importing in api).

2. Create `apps/web/src/features/analytics/lib/csv-export.ts`:
   - `exportToCsv(filename: string, headers: string[], rows: (string | number)[][]): void` — UTF-8 BOM prefix (`'\ufeff'`), comma-separated, cells containing commas/quotes/newlines wrapped in double quotes with internal quotes doubled. Creates Blob with `text/csv;charset=utf-8;` type, programmatic anchor click for download, then `URL.revokeObjectURL()`.

3. Create `apps/web/src/features/analytics/api/analytics-api.ts`:
   - Import supabase from `@/lib/supabase`.
   - `DailyMetricRow` interface matching daily_metrics columns: `{ metric_date: string, creative_id: string | null, campaign_id: string | null, impressions_served: number, impressions_viewable: number, clicks: number, engagements: number, video_plays: number, video_completes: number, total_dwell_time_ms: number }`.
   - `fetchDailyMetrics(advertiserId: string, startDate: string, endDate: string, filters?: AnalyticsFilters): Promise<DailyMetricRow[]>` — queries `daily_metrics` table with `.eq('advertiser_id', advertiserId).gte('metric_date', startDate).lte('metric_date', endDate).order('metric_date', { ascending: true })`. Applies optional `.eq('creative_id', filters.creativeId)` and `.eq('campaign_id', filters.campaignId)`.
   - `triggerTodayRollup(advertiserId: string): Promise<void>` — calls `supabase.rpc('rollup_today_metrics', { p_advertiser_id: advertiserId })`. Non-fatal: catches error and console.error only (dashboard still shows historical data).
   - `fetchCreativeOptions(advertiserId: string): Promise<{ id: string; name: string }[]>` — selects id, name from creatives where advertiser_id matches, ordered by name.
   - `fetchCampaignOptions(advertiserId: string): Promise<{ id: string; name: string }[]>` — selects id, name from campaigns where advertiser_id matches, ordered by name.

4. Create `apps/web/src/features/analytics/hooks/use-analytics.ts`:
   - Import useQuery from @tanstack/react-query, useAuth from @/contexts/auth-context.
   - `useAnalytics(startDate: string, endDate: string, filters?: AnalyticsFilters)` — queryKey: `['analytics', profile?.advertiser_id, startDate, endDate, filters]`. In queryFn: if endDate >= today's date string, call `triggerTodayRollup(profile.advertiser_id)` first, then `fetchDailyMetrics(...)`. `staleTime: 0` (ANLYT-05: fresh on reload). `enabled: !!profile?.advertiser_id`.
   - `useCreativeOptions()` — queryKey: `['creative-options', profile?.advertiser_id]`. Calls `fetchCreativeOptions`.
   - `useCampaignOptions()` — queryKey: `['campaign-options', profile?.advertiser_id]`. Calls `fetchCampaignOptions`.

Follow the billing feature module pattern (billing-api.ts, use-billing.ts) for consistent style.

# Plan 10-03: Analytics V2 — Interaction & Video Tracking

**Goal:** Close the "impressions-only" gap by implementing universal tracking in the renderer shell and updating all 14+ ad formats to report clicks, engagements, and video progress.

## Context
Phase 10-01 and 10-02 established the analytics dashboard and rollups. However, the creative renderers are not yet sending the `engagement` and `video` events defined in the schema. This plan upgrades the "V1" tracking to full "V2" fidelity.

## Tasks

1. **Task 1: Universal Tracking Helper in `renderer-shell.ts` (auto)**
   - Inject `window.stTrack(eventType, extraData)` into the iframe environment.
   - Automatically extract `rid`, `cid`, `aid`, and `cmpid` from the global `currentConfig`.
   - Use `navigator.sendBeacon` (where available) or a 1x1 image pixel to fire the event to the `/track-event` endpoint.
   - Implement a 5-second "heartbeat" to accumulate `dwell_time_ms`.

2. **Task 2: Upgrade Interactive Renderers (auto)**
   - Update `carousel`, `flipcard`, `scratch`, `quiz`, `cube`, `slider`, and `accordion` renderers.
   - Dispatch `engagement` events on meaningful user interactions (swipe, flip, scratch, select).

3. **Task 3: Upgrade Video Renderers (auto)**
   - Update `video-endcard` and `click-to-play` renderers.
   - Listen for video progress and dispatch `video_play`, `video_pause`, and quartiles (25, 50, 75, 100/complete).

4. **Task 4: Edge Function Verification (auto)**
   - Verify `supabase/functions/track-event/index.ts` is correctly logging these types and the `extra_data` payload.
   - Check `rollup_today_metrics` is processing them into the `daily_metrics` table.

## Verification
- Load an interactive creative in the editor.
- Perform interactions (swipe, flip, etc.).
- Confirm Network requests to `track-event` with correct query params.
- Confirm `extra_data` includes the interaction type.

1. **Add credit_balance column to advertisers:**
   ```sql
   ALTER TABLE public.advertisers ADD COLUMN IF NOT EXISTS credit_balance BIGINT NOT NULL DEFAULT 0;
   ```
   Default 0 means no credits purchased yet. BIGINT supports billions of impressions.

2. **Add rendered_html column to creatives:**
   ```sql
   ALTER TABLE public.creatives ADD COLUMN IF NOT EXISTS rendered_html TEXT;
   ```
   Stores the complete HTML document generated by `buildPreviewHtml()` at save time. Used by serve-ad Edge Function to avoid duplicating renderer code in Deno runtime.

3. **Add COMMENT on rendered_html** explaining its purpose (pre-rendered for ad serving, generated at save time).

4. **Create partial index for fast credit balance lookup during ad serving:**
   ```sql
   CREATE INDEX IF NOT EXISTS idx_advertisers_credit_balance ON public.advertisers (id) WHERE credit_balance > 0;
   ```

5. **Create atomic credit deduction function:**
   ```sql
   CREATE OR REPLACE FUNCTION public.deduct_impression_credit(p_advertiser_id UUID)
   RETURNS BOOLEAN
   LANGUAGE plpgsql
   SECURITY DEFINER
   SET search_path = public
   AS $$
   BEGIN
     UPDATE public.advertisers
     SET credit_balance = credit_balance - 1,
         updated_at = now()
     WHERE id = p_advertiser_id
       AND credit_balance >= 1;
     RETURN FOUND;
   END;
   $$;
   ```
   Uses SECURITY DEFINER with SET search_path per established pattern (Phase 1 decision). The `UPDATE...WHERE credit_balance >= 1` acquires implicit row-level lock preventing concurrent over-deduction. FOUND is true only if a row was updated.

6. **Add COMMENT on the function** explaining atomic deduction behavior.

Deploy with: `npx supabase db push --db-url "$DATABASE_URL"` (same pattern as Phase 1 -- no Docker dependency).

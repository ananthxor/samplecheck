# Plan 08-05: Ad Serving Updates for Self-Contained Bundles

**Goal:** Update the `serve-ad` edge function and the Tag Export UI to serve and correctly format the new self-contained Ad Tag Bundles. This ensures compatibility with GAM and other third-party servers.

## Context
Previously, `serve-ad` functioned by fetching the creative's `rendered_html` (which was an iframe shell) and injecting tracking macros via heavy string replacement. With the new `Ad Tag Bundler` from Phase 05-03, `rendered_html` is now a pre-compiled, self-contained payload that expects runtime configuration to initialize its embedded Telemetry engine.

## Tasks

### 1. Refactor `serve-ad` Edge Function
- Update `supabase/functions/serve-ad/index.ts`.
- Instead of performing string replacements on HTML, the edge function should output a minimal loader that:
  - Validates the request (CORS, origin, etc.).
  - Serves the creative's bundled payload.
  - Injects a script that initializes `window.ScrollTodaySDK.init()` with the specific `requestId`, `advertiserId`, `campaignId`, and base tracking URLs.

### 2. Update Tag Export Generation
- Update `apps/web/src/features/campaigns/components/tag-export-dialog.tsx` and `apps/web/src/features/campaigns/lib/tag-generator.ts`.
- Ensure the exported GAM/DFP and Direct Embed scripts are formatted correctly to load the new bundled script and pass any required macros (e.g., cache busters) through query parameters that the Telemetry engine expects.

### 3. Verify End-to-End Delivery
- Export a GAM tag for a Flipcard creative.
- Serve it via a local test HTML file or iframe.
- Verify that the format renders, the Telemetry engine initializes, and format-specific events (`flip`, `cta_clicked`) fire successfully back to the `track-event` edge function.

**Part A: track-event Edge Function**

Create `supabase/functions/track-event/index.ts` -- a GET+POST endpoint that receives tracking pixels/beacons and returns a 1x1 transparent GIF while inserting event data into ad_events.

1. Import `createAdminClient` from `../_shared/supabase-admin.ts` and `parseCookie`, `extractUtmParams`, `normalizeDevice` from `../_shared/tracking-utils.ts`.

2. Define `VALID_EVENTS` Set containing: `impression_served`, `impression_viewable`, `engagement`, `click`, `video_play`, `video_pause`, `video_complete`, `expand`, `collapse`, `close`.

3. Define `PIXEL` constant -- a 43-byte Uint8Array for a transparent 1x1 GIF:
   ```typescript
   const PIXEL = new Uint8Array([
     0x47, 0x49, 0x46, 0x38, 0x39, 0x61, 0x01, 0x00, 0x01, 0x00,
     0x80, 0x00, 0x00, 0xff, 0xff, 0xff, 0x00, 0x00, 0x00, 0x21,
     0xf9, 0x04, 0x01, 0x00, 0x00, 0x00, 0x00, 0x2c, 0x00, 0x00,
     0x00, 0x00, 0x01, 0x00, 0x01, 0x00, 0x00, 0x02, 0x02, 0x44,
     0x01, 0x00, 0x3b,
   ])
   ```

4. Define `PIXEL_HEADERS` with `Content-Type: image/gif`, `Cache-Control: no-store, no-cache, must-revalidate`, `Access-Control-Allow-Origin: *`.

5. In the `Deno.serve` handler:
   - Parse URL query params: `type` (event type), `rid` (request_id), `cid` (creative_id), `aid` (advertiser_id), `cmpid` (campaign_id)
   - If missing required params or invalid event type, return PIXEL immediately (never break ad rendering)
   - Build `extra_data` record from: cookie_id (parseCookie 'st_uid'), user_agent, device info (normalizeDevice), page_url from Referer header, UTM params (extractUtmParams)
   - Fire-and-forget insert to `ad_events` table: `admin.from('ad_events').insert({...}).then(() => {}).catch((err) => { console.error(...) })`
   - Return PIXEL response immediately (don't wait for DB insert)

**Part B: click-redirect Edge Function**

Create `supabase/functions/click-redirect/index.ts` -- a GET endpoint that logs a click event and 302 redirects to the destination URL.

1. Import `createAdminClient` from `../_shared/supabase-admin.ts` and `parseCookie` from `../_shared/tracking-utils.ts`.

2. In the `Deno.serve` handler:
   - Parse URL query params: `dest` (destination URL), `rid` (request_id), `cid` (creative_id), `aid` (advertiser_id), `cmpid` (campaign_id), `click` (optional GAM click URL)
   - If `dest` or `rid` is missing, return 400
   - Fire-and-forget insert to `ad_events`: event_type `click`, with extra_data containing destination_url, cookie_id, user_agent
   - Build final redirect URL: if `click` param present (GAM click URL), prepend it to dest: `gamClickUrl + encodeURIComponent(dest)`. Otherwise just use `dest`.
   - Return `Response(null, { status: 302, headers: { 'Location': finalUrl } })`

Both functions use `Deno.serve()` pattern (same as existing admin functions). Neither requires JWT verification -- they will be deployed with `--no-verify-jwt` since they receive requests from ad tags on publisher pages.

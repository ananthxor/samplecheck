<task type="auto">
  <name>Task 1: Build serve-ad Edge Function with tracking injection and credit gate</name>
  <files>supabase/functions/serve-ad/index.ts</files>
  <action>
Create `supabase/functions/serve-ad/index.ts` -- the main ad serving endpoint. This is a single large file containing the handler and all helper functions.

**Request handling (Deno.serve):**
1. Parse URL query params: `id` (creative_id, required), `w` (width, default '300'), `h` (height, default '250'), `cb` (cachebuster), `click` (GAM click URL from %%CLICK_URL_ESC%% macro)
2. If no `id`, return 204 (empty response -- silent fail for publisher page)

**Creative fetch:**
3. Use `createAdminClient()` to query creatives with a join to advertisers:
   ```
   .from('creatives')
   .select('id, advertiser_id, campaign_id, format_id, template_data, rendered_html, width, height, advertisers!inner(id, credit_balance)')
   .eq('id', creativeId)
   .eq('status', 'active')
   .single()
   ```
4. If error, no creative, or no `rendered_html` -- return 204

**Credit deduction:**
5. Call `admin.rpc('deduct_impression_credit', { p_advertiser_id: creative.advertiser_id })`
6. If returns false (balance zero) -- return 204

**Tracking injection (injectTracking helper function):**
7. Generate `requestId` via `crypto.randomUUID()`
8. Build base URLs for track-event and click-redirect from `url.origin + '/functions/v1/'`
9. Build impression pixel `<img>` tag with params: type=impression_served, rid, cid, aid, cmpid, cb=Date.now()
10. Build viewability script (see below)
11. Rewrite CTA click URLs in the rendered HTML: find `href="ORIGINAL_CTA_URL"` and replace with `href="CLICK_REDIRECT_URL?rid=...&cid=...&aid=...&cmpid=...&dest=ENCODED_ORIGINAL&click=ENCODED_GAM_CLICK"`. Add `target="_top"` so clicks open in parent window (not iframe).
12. Query `creative_trackers` joined with `tracker_configs` for this creative. For each tracker:
    - Replace `%%CACHEBUSTER%%` in tracker URL with `Date.now()`
    - `on_load` trackers: inject as `<img>` pixel alongside impression pixel
    - `on_viewable` trackers: inject as pixel fire inside the viewability callback (after fired=true)
    - `on_click` trackers: inject as `<img>` creation in a click handler on the creative root
    - `on_engagement` trackers: inject alongside engagement event handling
    - For `pixel` type: `<img src="URL" width="1" height="1" style="position:absolute;left:-9999px" alt="" />`
    - For `script` type: `<script src="URL"><\/script>`
13. Inject impression pixel + third-party on_load trackers + viewability script before `</body>`

**Viewability script (buildViewabilityScript helper):**
Create an IIFE that:
- Gets the `#creative-root` element
- If no IntersectionObserver, skip (graceful degradation)
- Creates IntersectionObserver with `threshold: [0, 0.5]`
- When `entry.isIntersecting && entry.intersectionRatio >= 0.5`: start a 1000ms setTimeout
- When not visible: clear the timeout
- On timeout fire (viewable): set `fired = true`, fire viewable impression pixel via `new Image().src`, fire on_viewable third-party tracker pixels, disconnect observer
- Listen for `visibilitychange` -- if `document.hidden`, clear timeout (pause timer on tab switch)

**Ad script builder (buildAdScript helper):**
Build the JavaScript string that creates an iframe on the publisher page:
- Use IIFE pattern `(function(){ ... })()`
- Get `document.currentScript.parentElement` as container
- Create iframe element with: width, height, frameBorder='0', scrolling='no', border:none, overflow:hidden
- Set `sandbox='allow-scripts allow-popups allow-popups-to-escape-sandbox allow-top-navigation-by-user-activation'`
- Set `allow='autoplay'` for video formats
- Set `srcdoc` to the tracking-injected HTML (escaped via `JSON.stringify()` to handle quotes, newlines, script tags)
- Append iframe to container

**Response:**
- Content-Type: `application/javascript` (NOT text/html -- Supabase rewrites text/html to text/plain on non-custom domains)
- Cache-Control: `no-store, no-cache, must-revalidate`
- Set-Cookie: `st_uid=UUID; Path=/; Max-Age=31536000; SameSite=None; Secure` (anonymous user tracking cookie, set or refresh)
- Access-Control-Allow-Origin: `*`

**Helper: escapeRegex(str)** -- Escape special regex characters for safe use in `new RegExp()`.

**Critical anti-patterns to avoid (from research):**
- Do NOT return text/html (Supabase rewrites it)
- Do NOT use document.write (blocked by modern browsers in async scripts)
- Do NOT include allow-same-origin in sandbox (security vulnerability)
- Do NOT include service_role key in the response (all tracking uses pixel/beacon requests)
- Do NOT include anon_key in the response

  <name>Task 2: Add anonymous user cookie and UTM/device context to serve-ad</name>
  <files>supabase/functions/serve-ad/index.ts</files>
  <action>
This task completes the DATA-04 identity/context layer within the serve-ad function (same file as Task 1, extending its functionality).

Ensure the serve-ad handler includes these identity/context features:

1. **Anonymous user cookie (st_uid):**
   - Read existing cookie from request: `parseCookie(req.headers.get('cookie'), 'st_uid')`
   - If no existing cookie, generate new UUID: `crypto.randomUUID()`
   - Set/refresh cookie in response headers: `Set-Cookie: st_uid=${cookieId}; Path=/; Max-Age=31536000; SameSite=None; Secure`
   - `SameSite=None; Secure` is required for cross-site cookie sending (ad on publisher page, cookie for Supabase domain)
   - Third-party cookies still work in Chrome (Google reversed deprecation July 2024, confirmed April 2025)
   - For browsers that block (Safari ITP, Firefox ETP), graceful degradation to request_id-only linking

2. **Pass cookie_id into tracking pixel URLs:**
   - Add `uid=${cookieId}` parameter to the impression pixel URL and viewability pixel URL so track-event can also receive it as a query param (backup to cookie)
   - track-event already reads cookies, but the query param provides a fallback

3. **Include context data in ad_events insert from serve-ad:**
   - serve-ad itself does NOT insert events (that's track-event's job), but ensure the impression pixel URL includes enough params for track-event to build full context
   - Params already planned: type, rid, cid, aid, cmpid, cb
   - The track-event function reads User-Agent, Referer, and cookies from the pixel request headers

4. **Verify the full identity chain works:**
   - serve-ad sets st_uid cookie on initial ad script response
   - Browser sends cookie back on subsequent requests to same domain (impression pixel, viewability pixel, click-redirect)
   - track-event reads cookie from request headers and stores in extra_data
   - click-redirect reads cookie from request headers and stores in extra_data
   - All events for same user linkable via cookie_id across sessions

Note: This task modifies the same file as Task 1. The implementation should be integrated into the serve-ad handler, not as a separate module. When implementing, ensure cookie handling is in the main handler response, and the cookie ID is available to the tracking URL builders.

  <action>
Create the migration file with two SECURITY DEFINER PL/pgSQL functions:

**fetch_hourly_metrics(p_advertiser_id UUID, p_date DATE)**
- Returns TABLE(hour_of_day INTEGER, impressions BIGINT, clicks BIGINT)
- Uses generate_series(0, 23) as base to guarantee all 24 hours are returned (zero-filled)
- LEFT JOINs event_counts on hour_of_day
- event_counts queries ad_events WHERE advertiser_id = p_advertiser_id AND event_timestamp >= p_date::TIMESTAMPTZ AND event_timestamp < (p_date + INTERVAL '1 day')::TIMESTAMPTZ
- Counts using FILTER: impressions counts event_type = 'impression_served', clicks counts event_type = 'click'
- Groups by EXTRACT(HOUR FROM ae.event_timestamp)::INTEGER
- Uses COALESCE(ec.impressions, 0) and COALESCE(ec.clicks, 0) for zero-filling
- SET search_path = public, SECURITY DEFINER

**fetch_device_breakdown(p_advertiser_id UUID, p_start_date DATE, p_end_date DATE)**
- Returns TABLE(device_type TEXT, impressions BIGINT)
- Queries ad_events WHERE advertiser_id = p_advertiser_id AND event_type = 'impression_served' AND event_timestamp >= p_start_date::TIMESTAMPTZ AND event_timestamp < (p_end_date + INTERVAL '1 day')::TIMESTAMPTZ
- Groups by COALESCE(ae.extra_data->>'device_type', 'unknown')
- Orders by impressions DESC
- Comment at top of function: "Scoped to impression_served events only for consistency with KPI impressions metric (ANLYT-11)"
- SET search_path = public, SECURITY DEFINER

Push migration using direct DB connection (NOT pooler URL — pooler gives "Tenant or user not found" on db push):

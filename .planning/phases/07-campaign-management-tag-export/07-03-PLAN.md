**Extend campaigns-api.ts** -- Add creative assignment functions (append to existing file from Plan 01):
- `fetchCreativesByCampaign(campaignId: string)`: Select creatives where campaign_id = campaignId, ordered by updated_at desc.
- `fetchUnassignedCreatives()`: Select creatives where campaign_id IS NULL, ordered by updated_at desc.
- `assignCreativeToCampaign(creativeId: string, campaignId: string)`: Update creative's campaign_id to campaignId. Return updated creative.
- `removeCreativeFromCampaign(creativeId: string)`: Update creative's campaign_id to null. Return updated creative.
- `updateCreativeStatus(creativeId: string, newStatus: Enums<'creative_status'>)`: Update creative's status field. Import and use `canCreativeTransitionTo` from status-machine.ts to validate the transition BEFORE making the API call. If transition is invalid, throw an Error with descriptive message. On success, return updated creative.

Also add corresponding React Query hooks to `use-campaigns.ts` (or create in campaigns hooks file):
- `useCampaignCreatives(campaignId: string | undefined)`: queryKey `['campaign-creatives', campaignId]`, enabled: !!campaignId
- `useUnassignedCreatives()`: queryKey `['unassigned-creatives']`
- `useAssignCreative()`: mutation, onSuccess invalidates `['campaign-creatives']`, `['unassigned-creatives']`, `['campaigns']`, and `['creatives']` (stale campaign creative count + creatives list).
- `useRemoveCreative()`: mutation, same invalidation as assign.
- `useUpdateCreativeStatus()`: mutation, onSuccess invalidates `['campaign-creatives']`, `['creatives']`.

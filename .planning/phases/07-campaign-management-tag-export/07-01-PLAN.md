Create the campaigns feature module API foundation following the exact pattern from `creatives-api.ts` and `use-creatives.ts`.

**campaigns-api.ts** -- Mirror creatives-api.ts exactly:
- `fetchCampaigns()`: Select all campaigns ordered by updated_at desc. RLS handles advertiser scoping automatically (no manual advertiser_id filter needed -- see Research pitfall 6).
- `fetchCampaignById(id)`: Select single campaign by id.
- `createCampaign(campaign: Insertable<'campaigns'>)`: Insert + select + single. Note: `advertiser_id` is required in the Insert type. The caller must provide it from auth context.
- `updateCampaign(id, updates: Updatable<'campaigns'>)`: Update + eq(id) + select + single.
- `deleteCampaign(id)`: Delete + eq(id).
- `fetchCampaignsWithCreativeCount()`: A more useful list query that includes the count of creatives per campaign. Use Supabase's ability to do: `supabase.from('campaigns').select('*, creatives(count)')`. This returns each campaign row with a `creatives` array containing `[{ count: N }]`. Transform the result to return `{ ...campaign, creative_count: number }[]`.

All functions use `import { supabase } from '@/lib/supabase'` and types from `@scrolltoday/shared`.

**use-campaigns.ts** -- Mirror use-creatives.ts exactly:
- `useCampaigns()`: useQuery with queryKey `['campaigns']`, queryFn calls `fetchCampaignsWithCreativeCount`.
- `useCampaign(id: string | undefined)`: useQuery with queryKey `['campaigns', id]`, enabled: !!id.
- `useCreateCampaign()`: useMutation, onSuccess invalidates `['campaigns']`.
- `useUpdateCampaign()`: useMutation accepting `{ id, updates }`, onSuccess invalidates `['campaigns']` and `['campaigns', id]`.
- `useDeleteCampaign()`: useMutation, onSuccess invalidates `['campaigns']`.

**status-machine.ts** -- Pure utility functions:
- Import `Enums` from `@scrolltoday/shared`.
- Define `CREATIVE_TRANSITIONS` record: draft -> [active], active -> [paused], paused -> [active, archived], archived -> [].
- Define `CAMPAIGN_TRANSITIONS` record: draft -> [active], active -> [paused], paused -> [active, completed], completed -> [].
- Export `getAvailableCreativeTransitions(current)`, `canCreativeTransitionTo(from, to)`.
- Export `getAvailableCampaignTransitions(current)`, `canCampaignTransitionTo(from, to)`.
- Both use the `Enums<'creative_status'>` and `Enums<'campaign_status'>` types respectively.

Important: Campaign status enum is `'draft' | 'active' | 'paused' | 'completed'` (NOT 'archived'). Creative status is `'draft' | 'active' | 'paused' | 'archived'` (NOT 'completed'). These are different enums -- see Research pitfall 1.
  </action>

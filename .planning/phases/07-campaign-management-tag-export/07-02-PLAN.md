**Migration file** `supabase/migrations/20260223000001_tracker_tables.sql`:

Create two tables following the exact pattern from the Research file's tracker schema section:

1. `tracker_configs` table:
   - `id UUID DEFAULT gen_random_uuid() PRIMARY KEY`
   - `advertiser_id UUID NOT NULL REFERENCES public.advertisers(id) ON DELETE CASCADE`
   - `name TEXT NOT NULL` (human-readable name like "DoubleVerify Viewability")
   - `tracker_url TEXT NOT NULL` (URL template, may contain macro placeholders)
   - `tracker_type TEXT NOT NULL CHECK (tracker_type IN ('pixel', 'script'))` -- pixel = 1x1 img, script = JS tag
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`

2. `creative_trackers` junction table:
   - `id UUID DEFAULT gen_random_uuid() PRIMARY KEY`
   - `creative_id UUID NOT NULL REFERENCES public.creatives(id) ON DELETE CASCADE`
   - `tracker_config_id UUID NOT NULL REFERENCES public.tracker_configs(id) ON DELETE CASCADE`
   - `fire_condition TEXT NOT NULL CHECK (fire_condition IN ('on_load', 'on_viewable', 'on_click', 'on_engagement'))`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `UNIQUE(creative_id, tracker_config_id, fire_condition)` -- prevent duplicate assignments

3. Indexes:
   - `idx_tracker_configs_advertiser ON tracker_configs (advertiser_id)`
   - `idx_creative_trackers_creative ON creative_trackers (creative_id)`
   - `idx_creative_trackers_config ON creative_trackers (tracker_config_id)`

4. RLS policies (enable RLS on both tables):
   - tracker_configs: Super admin full access (using `is_super_admin()` helper), Users manage own (using `get_user_advertiser_id()` helper). FOR ALL TO authenticated. Same pattern as campaigns RLS.
   - creative_trackers: Super admin full access. Users manage own via subquery checking creative ownership: `creative_id IN (SELECT id FROM creatives WHERE advertiser_id = get_user_advertiser_id())`.

5. Updated_at trigger on tracker_configs:
   - `CREATE TRIGGER set_updated_at BEFORE UPDATE ON public.tracker_configs FOR EACH ROW EXECUTE FUNCTION public.handle_updated_at();`
   - Note: `handle_updated_at()` function already exists from Phase 1 migration.

Deploy migration: `npx supabase db push --db-url "$DATABASE_URL"` (using the same pattern from Phase 1 decision 01-01).

**tag-generator.ts** -- Pure functions with zero dependencies:

Define `TagGeneratorInput` interface:
```typescript
interface TagGeneratorInput {
  creativeId: string
  width: number
  height: number
  serveBaseUrl: string // From VITE_SERVE_BASE_URL env var or window.location.origin fallback
}
```

`generateDfpTag(input: TagGeneratorInput): string`:
- Generate a DFP/GAM-compatible third-party ad tag.
- Include `%%CACHEBUSTER%%` macro for cache busting (CASE-SENSITIVE -- see Research pitfall 3).
- Include `%%CLICK_URL_ESC%%` macro for click tracking.
- Use `<ins>` wrapper element + async `<script>` tag pattern (modern standard, not synchronous document.write).
- Script src: `${serveBaseUrl}/serve/ad.js?id=${creativeId}&w=${width}&h=${height}&cb=%%CACHEBUSTER%%&click=%%CLICK_URL_ESC%%`
- Add HTML comment: `<!-- ScrollToday Ad Tag - DFP/GAM Compatible -->`
- Follow the exact format from the Research code examples section.

`generateEmbedTag(input: TagGeneratorInput): string`:
- Generate a self-contained embed tag for direct publisher placement.
- Use `<div>` wrapper with unique id `st-ad-${creativeId}`.
- Inline async `<script>` that creates and appends a script element with `Date.now()` cachebuster.
- Include `<noscript>` fallback with static image.
- Add HTML comment: `<!-- ScrollToday Ad - Direct Embed -->`
- Follow the exact format from the Research code examples section.

Export a helper `getServeBaseUrl(): string` that reads `import.meta.env.VITE_SERVE_BASE_URL` with fallback to `window.location.origin`.

**tracker-types.ts** -- Constants and Zod schemas:
- `FIRE_CONDITIONS = ['on_load', 'on_viewable', 'on_click', 'on_engagement'] as const`
- `type FireCondition = typeof FIRE_CONDITIONS[number]`
- `FIRE_CONDITION_LABELS: Record<FireCondition, string>` mapping to human-readable: "On Load", "On Viewable", "On Click", "On Engagement"
- `TRACKER_TYPES = ['pixel', 'script'] as const`
- `type TrackerType = typeof TRACKER_TYPES[number]`
- Zod schema for tracker config form validation: `trackerConfigSchema = z.object({ name: z.string().min(1), tracker_url: z.string().min(1).refine(url => url.startsWith('http'), { message: 'URL must start with http:// or https://' }), tracker_type: z.enum(['pixel', 'script']) })`. Note: use PERMISSIVE URL validation that allows macro placeholders like `%%CACHEBUSTER%%`, `${timestamp}`, `[random]` in the URL -- see Research pitfall 5. Do NOT use z.url() which would reject these.

**trackers-api.ts** -- Follow creatives-api.ts pattern:
- `fetchTrackerConfigs()`: Select all tracker_configs ordered by created_at desc. RLS handles scoping.
- `createTrackerConfig(config: Insertable<'tracker_configs'>)`: Insert + select + single.
- `updateTrackerConfig(id, updates: Updatable<'tracker_configs'>)`: Update + select + single.
- `deleteTrackerConfig(id)`: Delete by id.
- `fetchCreativeTrackers(creativeId: string)`: Select from creative_trackers where creative_id matches, with tracker_config joined: `supabase.from('creative_trackers').select('*, tracker_configs(*)').eq('creative_id', creativeId)`.
- `assignTrackerToCreative(creativeId, trackerConfigId, fireCondition)`: Insert into creative_trackers.
- `removeTrackerFromCreative(id)`: Delete creative_tracker by id.

**use-trackers.ts** -- React Query hooks following use-creatives.ts:
- `useTrackerConfigs()`: queryKey `['tracker-configs']`
- `useCreativeTrackers(creativeId: string | undefined)`: queryKey `['creative-trackers', creativeId]`, enabled: !!creativeId
- `useCreateTrackerConfig()`: invalidates `['tracker-configs']`
- `useUpdateTrackerConfig()`: invalidates `['tracker-configs']`
- `useDeleteTrackerConfig()`: invalidates `['tracker-configs']`
- `useAssignTracker()`: invalidates `['creative-trackers']` (all creative tracker queries)
- `useRemoveTracker()`: invalidates `['creative-trackers']`

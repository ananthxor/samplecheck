# Plan 05-03: Creative Format Bundler & Self-Contained Payloads

**Goal:** Transition interactive creative formats from the legacy `renderer-shell.ts` iframe-injection method to a self-contained Ad Tag Bundler. This ensures creatives carry their own format-specific interactive tracking and render perfectly within 3rd-party ad servers like GAM.

## Context
Currently, creatives are rendered via an iframe shell that globally monitors clicks. We need each format (Flipcard, Carousel, etc.) to be bundled with a unified Telemetry Engine so they can natively track format-specific events (e.g., `hover`, `face_1_clicked`, `swipe`) without external dependencies.

## Tasks

### 1. Create Standalone Telemetry Engine (`packages/ad-sdk/src/telemetry.ts`)
- Implement a vanilla JS `Telemetry` class that manages caching request contexts, firing pixels via `new Image().src`, and running viewability/dwell timers.
- Expose a global `window.ScrollTodaySDK.track(eventType, extraData)` method.
- Ensure this engine handles both generic events (impressions, clicks) and granular events from formats.

### 2. Implement the Creative Bundler (`packages/ad-sdk/src/bundler.ts`)
- Create a `buildAdPayload` function to replace `buildPreviewHtml` in `renderer-shell.ts`.
- It must stringify the configuration, inline the specific format's JS/CSS, and inject the new Telemetry logic into a single HTML document or Script tag payload.

### 3. Refactor Interactive Formats to Use Granular Tracking
- Update the JS payloads in `carousel`, `flipcard`, `cube`, etc., to trigger `window.ScrollTodaySDK.track()` for specific interactions instead of relying on the global click listener.
- **Example mappings:**
  - Flipcard: `track('flip')`, `track('cta_clicked')`
  - Carousel: `track('swipe', { slide_index: X })`
  - Cube: `track('cube_rotate', { face_index: X })`
  - Quiz: `track('quiz_answer', { choice: X, correct: true })`

### 4. Wire the Editor Preview
- Update `apps/web/src/features/editor/lib/renderer.ts` to utilize the new `buildAdPayload` from the `@scrolltoday/ad-sdk` so the editor preview matches the production output perfectly.

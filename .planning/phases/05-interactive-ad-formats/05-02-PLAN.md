**renderCube(root, cfg):**
- Guard: if `cfg.faces` length !== 4, call `renderPlaceholder` and return.
- Calculate `halfW = root.clientWidth / 2` for translateZ value (dynamic, responsive to ad size).
- Build HTML: `.cube-scene` > `.cube-container#cube-box` with 4 `.cube-face` divs. Each face gets inline `transform` style:
  - Face 0 (front): `rotateY(0deg) translateZ(${halfW}px)`
  - Face 1 (right): `rotateY(90deg) translateZ(${halfW}px)`
  - Face 2 (back): `rotateY(180deg) translateZ(${halfW}px)`
  - Face 3 (left): `rotateY(-90deg) translateZ(${halfW}px)`
- Each face renders: image if `f.imageUrl`, h2 with `f.headline`, p with `f.bodyText`.
- If `cfg.ctaText`, add `.sb-cta` anchor positioned absolutely at bottom center of the scene.
- Auto-rotate: set `frameTimer = setInterval(...)` that increments `currentFace = (currentFace + 1) % 4` and sets `cube.style.transform = 'rotateY(' + (-90 * currentFace) + 'deg)'` every `cfg.rotationSpeed || 4000` ms.
- Swipe gesture: add `pointerdown`/`pointerup` on `.cube-scene` for manual rotation. On swipe (`Math.abs(dx) > 30`), advance or reverse `currentFace`, clear and restart auto-rotate timer.

**renderScratch(root, cfg):**
- Build HTML: `.sc-container` > `.sc-reveal` (reveal image if `cfg.revealImageUrl`, h2 with headline, p with bodyText, `.sb-cta` anchor) + `<canvas id="scratch-canvas" class="sc-canvas"></canvas>`.
- Set `root.innerHTML`, then initialize canvas:
  - Get canvas and container references.
  - Scale for HiDPI: `canvas.width = container.clientWidth * (window.devicePixelRatio || 1)`, same for height. Set canvas CSS width/height to container dimensions.
  - Get 2D context, `ctx.scale(devicePixelRatio, devicePixelRatio)`.
  - If `cfg.overlayImageUrl`: create `new Image()`, set crossOrigin='anonymous', on load draw image filling container dimensions. As fallback before image loads, fill with `#c0c0c0` and add "Scratch here!" text centered.
  - If no overlayImageUrl: fill with solid `#c0c0c0`, add centered "Scratch here!" text.
- Pointer events on canvas:
  - `pointerdown`: `scratching = true`, `canvas.setPointerCapture(e.pointerId)`, call `scratch(e)`.
  - `pointermove`: if `scratching`, call `scratch(e)`.
  - `pointerup`: `scratching = false`, call `checkReveal()`.
- `scratch(e)` function: get position via `e.clientX - rect.left` (using `canvas.getBoundingClientRect()`). Set `ctx.globalCompositeOperation = 'destination-out'`. Draw filled circle at position with radius 20.
- `checkReveal()` function: call `ctx.getImageData(0, 0, canvas.width, canvas.height)`. Count pixels where alpha channel (`data[i+3]`) is 0. If cleared > 50% of total, fade out canvas: `canvas.style.transition = 'opacity 0.5s'; canvas.style.opacity = '0'`.
- **Important:** Only check reveal percentage on `pointerup` (not every move) to avoid getImageData performance issues per research pitfall 6.

**renderSlider(root, cfg):**
- Build HTML: `.sl-container` with:
  - Before image: `<img class="sl-img" src="..." />` (cfg.beforeImageUrl or empty)
  - After clip div: `.sl-after-clip` with after image inside (cfg.afterImageUrl or empty), initially `width:50%`
  - Handle: `.sl-handle` at left:50% with `.sl-handle-circle` inside containing `\u25C0\u25B6` (left-right arrows)
  - Labels: `.sl-label.sl-label-before` with "Before", `.sl-label.sl-label-after` with "After"
  - If `cfg.headline`, add a header overlay at top center
  - If `cfg.ctaText`, add `.sb-cta` at bottom center
- Set `root.innerHTML`, then attach pointer event listeners on `.sl-container`:
  - `pointerdown`: `dragging = true`, container.setPointerCapture(e.pointerId).
  - `pointermove`: if dragging, calculate `pct = ((e.clientX - containerRect.left) / containerRect.width) * 100`, clamped to `[5, 95]`. Update `.sl-after-clip` width, `.sl-handle` left position.
  - `pointerup`: `dragging = false`.

**renderQuiz(root, cfg):**
- Guard: if `cfg.options` empty/undefined, call `renderPlaceholder` and return.
- Build HTML: `.qz-container` > `#qz-question.qz-question` div with h2 (cfg.question), `.qz-options` div with `.qz-option` buttons (each with `data-idx` and `data-correct` attributes) + `#qz-result.qz-result` div (style `display:none`) with h2 (cfg.resultText) and `.sb-cta` anchor.
- Set `root.innerHTML`, then attach click listeners on each `.qz-option` button:
  - On click: disable all buttons. Add `selected` class to clicked button. Add `correct` class to buttons where `data-correct === 'true'`. Add `wrong` class to buttons where `data-correct !== 'true'`.
  - After 1000ms timeout: hide `#qz-question` (`display:none`), show `#qz-result` (`display:flex`).

**Important implementation notes (apply to ALL four renderers):**
- ALL user-provided strings MUST go through the existing `sanitize()` function.
- Use `var` declarations (not `let`/`const`) to match existing vanilla JS ES5 style.
- The `frameTimer` variable is shared -- cube auto-rotate uses it; it gets cleared at the top of `render()`.
- `touch-action:none` on scratch canvas and slider container to prevent browser scroll interference.

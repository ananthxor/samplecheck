  <name>Task 2: Implement carousel, flipcard, and accordion renderer functions and wire switch cases</name>

Add three renderer functions and wire them into the `render()` switch statement. All functions go inside the existing `<script>` block.

**1. Wire switch cases** -- Replace the `default: renderPlaceholder(root, cfg);` block with:
```
case 'carousel':  renderCarousel(root, cfg); break;
case 'flipcard':  renderFlipcard(root, cfg); break;
case 'accordion': renderAccordion(root, cfg); break;
default:          renderPlaceholder(root, cfg); break;
```

**2. renderCarousel(root, cfg):**
- Guard: if `cfg.slides` empty or undefined, call `renderPlaceholder(root, cfg)` and return.
- Build HTML: `.car-container` > `.car-track` with `.car-slide` divs for each slide (image if `s.imageUrl`, h2 from `s.headline`, p from `s.bodyText`). Add `.car-dots` div with `.car-dot` spans (first one gets `active` class).
- If `cfg.ctaText`, add `.sb-cta` anchor after the dots div (inside container), positioned with `style="display:block;text-align:center;padding:8px 0"` wrapping.
- Set `root.innerHTML` once, then attach event listeners.
- Swipe logic using Pointer Events on the track element:
  - `pointerdown`: record `startX = e.clientX`, set `dragging = true`, call `track.setPointerCapture(e.pointerId)`.
  - `pointerup`: if `dragging`, compute `dx = e.clientX - startX`. If `Math.abs(dx) > 30`, advance (`dx < 0` = next, `dx > 0` = prev), clamped to `[0, slides.length-1]`. Update `track.style.transform = 'translateX(-' + (current * 100) + '%)'`. Update dot active classes. Set `dragging = false`.
- Dot click: each dot gets a click listener that sets `current = dot.dataset.idx`, updates transform, updates active dot.
- Auto-play: if `cfg.autoPlay`, start `frameTimer = setInterval(...)` cycling slides every `cfg.autoPlayInterval || 3000` ms. On `pointerdown`, clear the interval. On `pointerup`, restart after 5 seconds via `setTimeout`.

**3. renderFlipcard(root, cfg):**
- Build HTML: `.fc-scene` > `.fc-card#fc-card` > `.fc-face.fc-front` (frontImageUrl if set, frontHeadline h2, frontBodyText p) + `.fc-face.fc-back` (backImageUrl if set, backHeadline h2, backBodyText p, `.sb-cta` anchor with ctaText).
- Use `sanitize()` for ALL user text values.
- Set `root.innerHTML`, then attach click listener on `.fc-scene` that toggles `is-flipped` class on `#fc-card`.

**4. renderAccordion(root, cfg):**
- Guard: if `cfg.sections` empty/undefined, call `renderPlaceholder(root, cfg)` and return.
- Build HTML: `.acc-container` with `.acc-section` divs for each section. Each section has `.acc-header` (sanitized title + `.acc-arrow` span with unicode `\u25BC`), and `.acc-body` > `.acc-body-inner` (sanitized content text + img if `s.imageUrl`).
- If `cfg.ctaText`, add `.sb-cta` anchor at bottom of container.
- Set `root.innerHTML`, then attach click listeners on each `.acc-header`.
- Click handler: toggle `open` class on the arrow span. If opening: set `.acc-body.style.maxHeight` to `.acc-body .acc-body-inner.scrollHeight + 'px'`. If closing: set `maxHeight = '0'`. Use `scrollHeight` of the inner element for accurate dynamic height.
- First section starts expanded (set maxHeight immediately after DOM insertion).

**Important implementation notes:**
- ALL user-provided strings MUST go through the existing `sanitize()` function.
- Use `var` declarations (not `let`/`const`) to match existing vanilla JS ES5 style in the renderer.
- The `frameTimer` variable is shared -- carousel auto-play uses it; it gets cleared at the top of `render()`.
- Add `touch-action:pan-y` on carousel track (already in CSS) to allow vertical scrolling while capturing horizontal swipes.

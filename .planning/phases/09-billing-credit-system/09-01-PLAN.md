Create `supabase/migrations/20260225000001_billing_tables.sql` with:

1. **credit_transactions table** -- append-only ledger:
   - `id UUID DEFAULT gen_random_uuid() PRIMARY KEY`
   - `advertiser_id UUID NOT NULL REFERENCES public.advertisers(id) ON DELETE CASCADE`
   - `type TEXT NOT NULL CHECK (type IN ('purchase', 'adjustment', 'refund'))` -- TEXT with CHECK, same pattern as tracker_type in Phase 7
   - `amount BIGINT NOT NULL` -- positive for additions, negative for deductions
   - `balance_after BIGINT` -- snapshot of balance after this transaction
   - `stripe_session_id TEXT UNIQUE` -- Stripe Checkout Session ID for idempotency
   - `stripe_payment_intent_id TEXT` -- Stripe PaymentIntent ID
   - `receipt_url TEXT` -- Stripe-hosted receipt URL
   - `metadata JSONB DEFAULT '{}'`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - Index: `CREATE INDEX idx_credit_transactions_advertiser ON public.credit_transactions (advertiser_id, created_at DESC)`
   - RLS enabled with same pattern as other tables:
     - "Super admin full access to transactions" policy (FOR ALL, USING is_super_admin(), WITH CHECK is_super_admin())
     - "Users can read own transactions" policy (FOR SELECT, USING advertiser_id = get_user_advertiser_id())
   - No updated_at trigger -- transactions are append-only/immutable

2. **add_impression_credits function**:
   - `CREATE OR REPLACE FUNCTION public.add_impression_credits(p_advertiser_id UUID, p_amount BIGINT) RETURNS BIGINT`
   - LANGUAGE plpgsql, SECURITY DEFINER, SET search_path = public (matches Phase 1 pattern)
   - UPDATE advertisers SET credit_balance = credit_balance + p_amount, updated_at = now() WHERE id = p_advertiser_id RETURNING credit_balance INTO v_new_balance
   - IF NOT FOUND THEN RAISE EXCEPTION
   - RETURN v_new_balance

3. **Fix daily_metrics UNIQUE constraint** for NULL campaign_id handling:
   - `ALTER TABLE public.daily_metrics DROP CONSTRAINT IF EXISTS daily_metrics_metric_date_advertiser_id_campaign_id_creative_key;`
   - `CREATE UNIQUE INDEX IF NOT EXISTS idx_daily_metrics_unique ON public.daily_metrics (metric_date, advertiser_id, campaign_id, creative_id) NULLS NOT DISTINCT;`

4. **rollup_daily_metrics function**:
   - `CREATE OR REPLACE FUNCTION public.rollup_daily_metrics(p_date DATE DEFAULT (CURRENT_DATE - INTERVAL '1 day')::DATE) RETURNS INTEGER`
   - LANGUAGE plpgsql, SECURITY DEFINER, SET search_path = public
   - INSERT INTO daily_metrics ... SELECT ... FROM ad_events WHERE event_timestamp >= p_date AND < p_date + 1 day, GROUP BY advertiser_id, campaign_id, creative_id
   - ON CONFLICT (metric_date, advertiser_id, campaign_id, creative_id) DO UPDATE SET all counters to EXCLUDED values, updated_at = now()
   - Use CTE with RETURNING to count rows upserted
   - Follow exact pattern from research Pattern 5

5. **pg_cron setup**:
   - `CREATE EXTENSION IF NOT EXISTS pg_cron WITH SCHEMA pg_catalog;`
   - `GRANT USAGE ON SCHEMA cron TO postgres;`
   - `SELECT cron.schedule('rollup-daily-metrics', '0 2 * * *', $$SELECT public.rollup_daily_metrics()$$);`
   - NOTE: pg_cron must be enabled in Supabase Dashboard BEFORE migration runs. Include a comment in the SQL stating this prerequisite.

Create `supabase/functions/_shared/stripe.ts`:
- `import Stripe from 'https://esm.sh/stripe@14?target=denonext'` (exact import from Supabase official example)
- `export function createStripeClient()` returning `new Stripe(Deno.env.get('STRIPE_API_KEY')!, { apiVersion: '2024-11-20' })`
- `export const cryptoProvider = Stripe.createSubtleCryptoProvider()`
- Export `CREDIT_PACKS` config object with 3 packs: `'50k' -> { credits: 50000, label: '50,000 Impressions', priceId: Deno.env.get('STRIPE_PRICE_50K')! }`, `'200k' -> { credits: 200000, ... }`, `'1m' -> { credits: 1000000, ... }`
- Export `type PackId = keyof typeof CREDIT_PACKS`

Create `supabase/functions/create-checkout/index.ts`:
- Authenticated endpoint (verifies JWT via `auth.getUser(jwt)` -- same pattern as admin functions)
- CORS preflight handler (OPTIONS returns corsHeaders)
- Steps:
  1. Extract and verify JWT from Authorization header using `createAdminClient().auth.getUser(jwt)`
  2. Get advertiser_id from user_profiles via auth_user_id lookup
  3. Parse `{ packId }` from request body, validate against CREDIT_PACKS
  4. Get advertiser contact_email for Stripe customer_email field
  5. Create Stripe Checkout Session with:
     - `mode: 'payment'`
     - `line_items: [{ price: pack.priceId, quantity: 1 }]`
     - `success_url: origin/billing?success=true&session_id={CHECKOUT_SESSION_ID}`
     - `cancel_url: origin/billing?canceled=true`
     - `client_reference_id: advertiser_id`
     - `customer_email: advertiser.contact_email || user.email`
     - `metadata: { advertiser_id, credits: String(pack.credits), pack_id: packId }`
  6. Return `{ url: session.url }` with CORS headers
- Error handling: 401 for auth failures, 400 for bad pack selection, with JSON error body

Create `supabase/functions/stripe-webhook/index.ts`:
- Public endpoint (deployed with --no-verify-jwt, no CORS needed since Stripe calls directly)
- Steps:
  1. Read body with `await req.text()` (NOT req.json() -- critical for signature verification)
  2. Get Stripe-Signature header
  3. Verify signature via `stripe.webhooks.constructEventAsync(body, signature, STRIPE_WEBHOOK_SIGNING_SECRET, undefined, cryptoProvider)`
  4. If event.type === 'checkout.session.completed' and session.payment_status === 'paid', call fulfillCreditPurchase()
  5. Always return `{ received: true }` with status 200
- `fulfillCreditPurchase(session)`:
  1. Extract advertiser_id and credits from session.metadata
  2. Idempotency check: query credit_transactions for existing stripe_session_id, return early if found
  3. Call `admin.rpc('add_impression_credits', { p_advertiser_id, p_amount: credits })` to atomically add credits
  4. Insert into credit_transactions: advertiser_id, type='purchase', amount=credits, balance_after from RPC return, stripe_session_id, stripe_payment_intent_id, metadata with pack_id and customer_email
  5. Log success

Deploy both Edge Functions:
- `npx supabase functions deploy create-checkout` (uses JWT verification by default -- do NOT add --no-verify-jwt)
- `npx supabase functions deploy stripe-webhook --no-verify-jwt` (public endpoint for Stripe to call)

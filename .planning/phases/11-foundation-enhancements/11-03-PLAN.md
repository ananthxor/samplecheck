1. Create `apps/web/src/features/trackers/components/tracker-table.tsx`:

A table component that displays tracker configs with filtering and search.

Props: `trackers: Tables<'tracker_configs'>[]`, `onEdit: (tracker) => void`, `onDelete: (id: string) => void`

Implementation:
- Local state: `activeCategory` (string, default "all"), `search` (string, default "")
- Category filter tabs: render a row of `Button` components (variant "outline" for inactive, "default" for active) for ["All", "Conversion", "Impression", "Click"]. Clicking a tab sets `activeCategory`.
- Search: an `Input` with search icon (Search from lucide) and placeholder "Search trackers..."
- Filtering logic (client-side):
  ```typescript
  const filtered = trackers.filter(t => {
    const matchesCategory = activeCategory === 'all' || t.category === activeCategory
    const matchesSearch = t.name.toLowerCase().includes(search.toLowerCase())
    return matchesCategory && matchesSearch
  })
  ```
- Render a shadcn Table with columns:
  - **Name**: `t.name`
  - **Category**: badge with category label (use `Badge` component, variant based on category)
  - **Type**: `TRACKER_TYPE_LABELS[t.tracker_type]`
  - **URL**: truncated URL (show first 50 chars + "...")
  - **Actions**: dropdown menu with Edit and Delete options (MoreHorizontal icon trigger)
- Empty state: "No trackers found." message when filtered list is empty
- Show count: "{filtered.length} tracker(s)" below the table

Do NOT use @tanstack/react-table here -- the tracker list is simpler and a static shadcn Table with manual filtering is sufficient and keeps the plan lighter.

1. In `campaigns-api.ts`, add a new `CampaignTableRow` type and `fetchCampaignsForTable` function:

```typescript
export type CampaignTableRow = Tables<'campaigns'> & {
  creative_count: number
  impressions_served: number
}
```

`fetchCampaignsForTable` should:
- Fetch all campaigns with creative count (same as existing `fetchCampaignsWithCreativeCount`)
- Also fetch `daily_metrics` for the advertiser, selecting only `campaign_id` and `impressions_served` columns, filtered to rows where `campaign_id IS NOT NULL`
- Aggregate impressions_served per campaign_id client-side using a Map
- Return `CampaignTableRow[]` merging campaigns with their aggregated impressions

The function needs the advertiser_id parameter. Get it the same way other API functions do -- the existing pattern uses RLS so just fetch all campaigns (RLS scopes to user's advertiser). For daily_metrics, also just fetch all (RLS scopes). No explicit advertiser_id param needed.

```typescript
export async function fetchCampaignsForTable(): Promise<CampaignTableRow[]> {
  // Fetch campaigns with creative count
  const { data: rawCampaigns, error: campError } = await supabase
    .from('campaigns')
    .select('*, creatives(count)')
    .order('updated_at', { ascending: false })

  if (campError) throw new Error(campError.message)

  // Fetch daily_metrics impressions grouped by campaign
  const { data: metrics, error: metError } = await supabase
    .from('daily_metrics')
    .select('campaign_id, impressions_served')
    .not('campaign_id', 'is', null)

  if (metError) throw new Error(metError.message)

  // Aggregate impressions per campaign
  const impressionMap = new Map<string, number>()
  for (const row of metrics ?? []) {
    if (row.campaign_id) {
      impressionMap.set(
        row.campaign_id,
        (impressionMap.get(row.campaign_id) ?? 0) + row.impressions_served
      )
    }
  }

  return rawCampaigns.map((campaign) => {
    const { creatives, ...rest } = campaign as Tables<'campaigns'> & {
      creatives: { count: number }[]
    }
    return {
      ...rest,
      creative_count: creatives?.[0]?.count ?? 0,
      impressions_served: impressionMap.get(rest.id) ?? 0,
    }
  })
}
```

Keep the existing `fetchCampaignsWithCreativeCount` function (other code may use it) but the campaign list will now use `fetchCampaignsForTable`.

**JS section** -- rewrite `renderCountdown(root, cfg)`:
1. Parse target date: `var target = new Date(cfg.targetDate).getTime();` -- if `isNaN(target)`, call `renderPlaceholder(root, cfg); return;` (guard against invalid/empty date)
2. Build HTML with headline, body text, timer display (4 unit blocks with span IDs: `cd-days`, `cd-hours`, `cd-mins`, `cd-secs`), each with a `.cd-value` span and `.cd-label` span, separated by `.cd-sep` colon dividers, and CTA button
3. Define `tick()` function:
   - `var diff = Math.max(0, target - Date.now());`
   - Calculate days: `Math.floor(diff / 86400000)`
   - Calculate hours: `Math.floor((diff % 86400000) / 3600000)`
   - Calculate minutes: `Math.floor((diff % 3600000) / 60000)`
   - Calculate seconds: `Math.floor((diff % 60000) / 1000)`
   - Update each span using `document.getElementById('cd-days')` etc. with `String(n).padStart(2, '0')`
   - Guard each getElementById with null check (if element exists, update textContent)
   - If `diff <= 0`, call `clearInterval(frameTimer)` to stop ticking
4. Call `tick()` immediately for first render (no flash of 00:00:00:00 before first interval fires)
5. Set `frameTimer = setInterval(tick, 1000)` -- this uses the global `frameTimer` variable from renderer-shell.ts which gets cleaned up automatically on re-render

The `cfg.targetDate` field is an ISO date string from the date picker in config.ts. The default template sets it to 7 days from now.

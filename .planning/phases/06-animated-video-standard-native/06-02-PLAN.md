**renderer-shell.ts** -- In the `render(cfg)` function inside the generated HTML script string, add video cleanup AFTER the existing `if (frameTimer) { clearInterval(frameTimer); frameTimer = null; }` block and BEFORE the format render call. Add these lines:

```javascript
// Cleanup video elements (prevent ghost audio on re-render)
var videos = root.querySelectorAll('video');
for (var v = 0; v < videos.length; v++) {
  videos[v].pause();
  videos[v].removeAttribute('src');
  videos[v].load();
}
```

The `removeAttribute('src')` + `load()` pattern ensures the browser releases the media resource. This is defensive -- not all formats use video, but the cleanup is zero-cost when no video elements exist.

**editor-preview.tsx** -- On the `<iframe>` element (currently has `sandbox="allow-scripts"`), add `allow="autoplay"` as a separate attribute. The JSX should look like:
```jsx
<iframe
  ref={iframeRef}
  srcDoc={initialHtml}
  sandbox="allow-scripts"
  allow="autoplay"
  title="Creative preview"
  ...
/>
```

**preview-page.tsx** -- On the `<iframe>` element (line ~80, currently has `sandbox="allow-scripts"`), add `allow="autoplay"` as a separate attribute, same pattern as editor-preview.

These two attributes serve different purposes: `sandbox` restricts capabilities, `allow` grants specific permissions. Both are needed for muted video autoplay inside sandboxed iframes.
